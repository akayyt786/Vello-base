CORE FEATURES
================================================================================

1. Collections
   - Hierarchical paths supporting subcollections
   - Schema metadata storage
   - Document count tracking
   - Full REST CRUD

2. Documents
   - JSONB data storage (arbitrary nested data)
   - Firestore-compatible field types
   - Automatic timestamps
   - Optimistic locking via __v version counter

3. Queries
   - 10 operators: ==, !=, <, <=, >, >=, in, not-in, array-contains, array-contains-any
   - Multiple WHERE conditions (AND logic)
   - Multi-field ORDER BY (ASC/DESC)
   - Cursor-based keyset pagination

4. Transactions
   - Atomic batch write operations (set, update, delete)
   - All-or-nothing semantics
   - Detailed per-operation results

5. Security & Multi-Tenancy
   - Project-scoped isolation
   - JWT authentication
   - Permission checks (IsAuthenticated, IsProjectMember)
   - Audit trail (created_by, updated_by)

6. Performance
   - GIN index on JSONB data
   - Expression indexes for common queries
   - Cursor pagination (O(k) not O(n+k))
   - No offset overhead

================================================================================
DATABASE SCHEMA
================================================================================

Collections Table:
  - id (UUID PK)
  - project_id (FK)
  - name, path, schema (JSONB), document_count
  - created_at, updated_at
  - created_by_id, updated_by_id (audit)
  - Unique: (project, path)
  - Indexes: (project, name), (project, created_at)

Documents Table:
  - id (UUID PK)
  - project_id (FK)
  - collection_path, doc_id
  - data (JSONB)
  - __v (version counter)
  - created_at, updated_at
  - created_by_id, updated_by_id (audit)
  - Unique: (project, collection_path, doc_id)
  - Indexes: GIN(data), (project, collection_path, created_at), ...

QUERY OPERATORS & EXAMPLES
================================================================================

Equality:
  {"field": "status", "op": "==", "value": "active"}

Comparison:
  {"field": "age", "op": ">", "value": 18}
  {"field": "price", "op": "<=", "value": 100}

Membership:
  {"field": "status", "op": "in", "value": ["active", "pending"]}
  {"field": "status", "op": "not-in", "value": ["deleted"]}

Arrays:
  {"field": "tags", "op": "array-contains", "value": "admin"}
  {"field": "tags", "op": "array-contains-any", "value": ["admin", "editor"]}

Order By:
  {"field": "created_at", "direction": "desc"}
  {"field": "name", "direction": "asc"}

USAGE EXAMPLE
================================================================================

# Create collection
curl -X POST http://localhost:8000/api/projects/proj123/collections/ \
  -H "Authorization: Bearer {token}" \
  -d '{"name":"users", "path":"users"}'

# Create document
curl -X POST http://localhost:8000/api/projects/proj123/collections/users/docs/ \
  -H "Authorization: Bearer {token}" \
  -d '{"doc_id":"alice", "data":{"name":"Alice","age":30}}'

# Query documents (filter + sort)
curl "http://localhost:8000/api/projects/proj123/collections/users/docs/" \
  -H "Authorization: Bearer {token}" \
  --get --data-urlencode 'query={"where":[{"field":"age","op":">","value":25}],"orderBy":[{"field":"name","direction":"asc"}]}'

# Batch write transaction
curl -X POST http://localhost:8000/api/projects/proj123/transaction/ \
  -H "Authorization: Bearer {token}" \
  -d '{
    "writes": [
      {"op":"set", "path":"users/bob", "data":{"name":"Bob"}},
      {"op":"delete", "path":"users/charlie"}
    ]
  }'

PERFORMANCE CHARACTERISTICS
================================================================================

---------------------|-----------------|-----
Create document      | O(1)            | Single insert
Get document         | O(1)            | Primary key lookup
Update document      | O(1)            | Direct update
Delete document      | O(1)            | Direct delete
List documents       | O(n)            | n = result set size
Query with filters   | O(n)            | Uses JSONB indexes
Cursor pagination    | O(k)            | k = page size; no full scan

  2. Version conflict on update
     → Implement retry logic with exponential backoff

     → Check indexes exist for filtered fields
