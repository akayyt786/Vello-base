================================================================================
RUNNING THE TESTS
================================================================================

1. Install Dependencies
   $ pip install -r requirements.txt

2. Setup Database
   $ python manage.py migrate

3. Run All Tests
   $ pytest tests/ -v

4. Run With Coverage
   $ pytest tests/ -v --cov=core,data,rules --cov-report=html

5. Run Specific Test File
   $ pytest tests/test_auth.py -v
   $ pytest tests/test_data_api.py -v
   $ pytest tests/test_rules.py -v

6. Run Specific Test
   $ pytest tests/test_auth.py::TestAuthViewSet::test_login_success -v

7. Run Tests Matching Keyword
   $ pytest tests/ -k "auth" -v
   $ pytest tests/ -k "query" -v
   $ pytest tests/ -k "transaction" -v

8. Run Smoke Tests (Integration)
   $ python manage.py runserver
   $ bash tests/curl_smoke_test.sh              # In another terminal
   $ bash tests/curl_smoke_test.sh http://example.com:8000

9. Run With Debugging
   $ pytest tests/ -v --pdb                     # PDB on failure
   $ pytest tests/ -vv --tb=long                # Detailed traceback

10. Generate Coverage Report
    $ pytest tests/ --cov=core,data,rules --cov-report=html
    $ open htmlcov/index.html                   # View in browser

================================================================================
TEST EXECUTION FLOW
================================================================================

PYTEST Test Flow:
1. Load conftest.py fixtures
2. Create test database (Django ORM)
3. For each test:
   a. Setup fixtures (create test_user, test_project, etc.)
   b. Execute test method (with @pytest.mark.django_db)
   c. Assert response status and data
   d. Assert database state changes
   e. Cleanup (rollback transaction)

CURL Smoke Test Flow:
1. Register new user → get JWT tokens
2. Login with credentials → verify tokens
3. Get current user profile
4. Refresh access token
5. Create project
6. Create collection
7. Create document (alice)
8. Read document
9. Update document (age 30 → 31)
10. List documents
11. Create additional documents (user1, user2, user3)
12. Query documents with filter
13. Delete document (user1)
14. Register second user → test unauthorized access
15. Verify second user cannot access first user's project (403)
16. Test version conflict on concurrent update (409)
17. Transaction: batch write (bob, charlie)
18. Logout and verify token blacklist
