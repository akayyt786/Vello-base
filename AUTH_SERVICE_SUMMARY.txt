KEY FEATURES

1. Email/Password Authentication
   - Register with email + password
   - Login returns JWT pair (access + refresh)
   - Passwords validated against Django's password validators

2. JWT Tokens
   - Access tokens: 15 minutes (short-lived, low risk)
   - Refresh tokens: 7 days (long-lived)
   - Algorithm: HS256 with Django SECRET_KEY
   - Token rotation: Old refresh token blacklisted on renewal
   - Custom claims embedded in access token payload

3. Custom Claims
   - Store user roles, permissions, metadata in token
   - Max 1000 bytes (Firebase-compatible)
   - Set via /api/auth/set-custom-claims/ (admin only)
   - Appear in JWT immediately (no propagation delay)
   - Example: { "admin": true, "role": "editor", "tier": "premium" }

4. Token Blacklist / Logout
   - Logout blacklists refresh token
   - Prevents token reuse after logout
   - Stored in RefreshTokenBlacklist model
   - CustomJWTAuthentication checks blacklist on every request

5. Anonymous Sign-in
   - Create temporary user without email/password
   - Useful for guest sessions before account creation
   - Can be upgraded to full account in Phase 2

6. User Profiles
   - Email verification status
   - Sign-in provider (password, google, github, anonymous)
   - Phone number + verification
   - Avatar URL, bio
   - Custom claims (JSON)

7. Multi-Tenant Support
   - Project-based isolation
   - project_id passed in JWT claims during login
   - Used by MultiTenantMiddleware for context/RLS

TESTING THE IMPLEMENTATION

Run all tests:
    pytest tests/test_auth.py -v

Run specific test:
    pytest tests/test_auth.py::TestAuthViewSet::test_register_success -v

With coverage:
    pytest tests/test_auth.py --cov=api --cov=core --cov-report=html

Command-line testing (after server running on :8000):

1. Register:
curl -X POST http://localhost:8000/api/auth/register/ \
  -H "Content-Type: application/json" \
  -d '{
    "email":"test@test.com",
    "password":"SecurePass123!",
    "password_confirm":"SecurePass123!"
  }'

2. Login:
curl -X POST http://localhost:8000/api/auth/login/ \
  -H "Content-Type: application/json" \
  -d '{
    "email":"test@test.com",
    "password":"SecurePass123!"
  }'

3. Get profile with token:
curl -X GET http://localhost:8000/api/auth/me/ \
  -H "Authorization: Bearer <ACCESS_TOKEN>"
